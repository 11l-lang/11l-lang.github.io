Целочисленные литералы



print(255)        // десятичное число
print(0000'00FF)  // шестнадцатеричное число
print(00'FF)      // короткое шестнадцатеричное число
print(F'F)        // сверхкороткое (однобайтовое) шестнадцатеричное число
print(377o)       // восьмеричное число
print(1111'1111b) // двоичное число
print(255'000)    // десятичное число
print(1'00000000) // специальное шестнадцатеричное число
Вывод:
255
255
255
255
255
255
255000
4294967296

Обратите внимание, что разделитель цифр 
'
обязателен для шестнадцатеричных чисел.
[Кто-то может захотеть вставить в код номер кредитной карты (например 1234'5678'9012'3456, что является шестнадцатеричным (а не десятичным!) числом в 11l), но насколько часто это требуется на практике?
[{…
Т.о. шестнадцатеричные числа как правило должны состоять из числа байт степени двойки [а в этом случае в 11l требуется всего один апостроф 
'
].

Хотя признаю, что такая форма записи шестнадцатеричных чисел/литералов/констант немного непривычна.
Но оттого и интересно, сможет ли мозг адаптироваться к такой записи. Можно считать, что это эксперимент такой.

Также такая запись поощряет красиво писать шестнадцатеричные числа (в C++14 можно писать и 
0xFFFF'FFFF
и 
0xFFFFFFFF
, но последнюю запись можно спутать с 
0xFFFFFFF
{…вообще, запись 
0xFFFFFFF
не имеет смысла: должно быть либо 
0x0FFFFFFF
, либо 
0xFFFFFFFF
} или 
0xFFFFFFFFF
{…а с 
0xFFFFFFFFFFFFFFFF
совсем плохо без разделителей} — обратите внимание, что ни запись 
FFF'FFFF
, ни 
FFFFF'FFFF
— некорректны в 11l — необходимо писать 
0FFF'FFFF
или 
0000000F'FFFFFFFF
).

В крайнем случае можно добавить суффикс 
h
\
ш
для обозначения шестнадцатеричных чисел (сохранив при этом обязательные разделители).
}]]

Почему 
'
, а не 
_
:
  1. Может быть действительное имя идентификатора (например, имя перечислителя 
    C0_FE
    ).
  2. Даже в простом тексте иногда встречается 
    '
    как разделитель разрядов (вот или вот), но никогда не встречается 
    _
    .

L

Long
