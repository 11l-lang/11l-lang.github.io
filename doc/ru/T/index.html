Почему T/type?
Я согласен с Никлаусом Виртом:
https://www.osp.ru/os/1996/06/179017:
Мы также избегаем введения совершенно новых терминов для хорошо известных понятий; например, мы остаемся верными термину "тип", избегая слова "класс"

К тому же, если посмотреть на определение термина «класс» в C++ {…
  • https://en.cppreference.com/w/cpp/language/classes:

    A class is a user-defined type.
  • Standard for Programming Language C++:

    A class is a type.
}, то видно, что класс — это пользовательский тип.
